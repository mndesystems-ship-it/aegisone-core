Which first-use layout feels easiest?

The goal is that someone can open the page cold and still know what to do next.

A. Guided Check
MNDe Proof Check Endpoint ready
1. Choose a sample or import JSON
2. Run decision check
3. Review result and receipt

Guided Check

Turns the app into a simple three-step task. Best for people who do not know what MNDe, receipts, or policies mean yet.

B. Plain Dashboard
Request | Result | Receipt | System
Paste or import request
Decision summary
Receipt tools

Plain Dashboard

Keeps all current tools visible, but renames technical labels and groups them by job.

C. Receipt First
Drop a receipt to verify it
Result explained in plain language

Receipt First

Optimized for auditors who arrive with a receipt file and only need to prove whether it is valid.

My recommendation

Pick Guided Check. It gives the page a clear beginning, middle, and end, while keeping the receipt tools available after a result exists.
